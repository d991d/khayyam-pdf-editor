# Khayyam PDF Editor

A native macOS PDF editor built with SwiftUI and MuPDF, developed by [d991d](https://github.com/d991d).

## Features

- **Annotate** — highlight, underline, strikethrough, sticky notes, freehand ink
- **Draw** — rectangles, ovals, and lines with customizable stroke/fill
- **Typewriter** — click anywhere on a page to add floating text
- **Edit existing text** — click any text block to modify PDF content in place (powered by MuPDF)
- **Insert images** — drag an image from disk onto any page
- **Merge PDFs** — combine multiple PDF files into one
- **Split PDF** — extract page ranges into separate files
- **Rotate pages** — rotate individual pages left or right
- **Zoom controls** — zoom in/out, fit to page, actual size
- **Keyboard shortcuts** — full menu-driven workflow

## Requirements

| | Minimum |
|---|---|
| macOS | 13.0 Ventura |
| Architecture | Apple Silicon (arm64) |
| Xcode | 15 or later |

> Intel Macs are not supported because MuPDF is built as an arm64-only static library. To add Intel support, build MuPDF from source as a universal binary.

## Getting Started

### 1. Clone the repository

```bash
git clone https://github.com/d991d/khayyam-pdf-editor.git
cd khayyam-pdf-editor
```

### 2. Run setup

```bash
./setup.sh
```

This script will:
- Install [XcodeGen](https://github.com/yonaskolb/XcodeGen) via Homebrew (if not already installed)
- Check for MuPDF (install it or detect a source build)
- Generate `Config/LocalPaths.xcconfig` with the correct Homebrew paths for your machine
- Generate `KhayyamPDFEditor.xcodeproj` via XcodeGen
- Open the project in Xcode

### 3. Build in Xcode

1. Select your **Team** in **Signing & Capabilities**
2. Press **⌘B** to build
3. Press **⌘R** to run

## MuPDF Dependency

This app uses [MuPDF](https://mupdf.com) for PDF text editing. Homebrew ships MuPDF as a dynamic library, which is rejected by the macOS hardened runtime. For distribution, you must build MuPDF from source as a static library:

```bash
# Download MuPDF 1.27.2 source
curl -L https://mupdf.com/downloads/archive/mupdf-1.27.2-source.tar.gz -o /tmp/mupdf.tar.gz
tar -xf /tmp/mupdf.tar.gz -C /tmp
cd /tmp/mupdf-1.27.2-source

# Build and install static libraries
make HAVE_X11=no HAVE_GLUT=no
sudo make install HAVE_X11=no HAVE_GLUT=no
```

After installing, re-run `./setup.sh` — it will automatically detect the static libraries and configure the project.

## Architecture

The app follows an MVVM pattern:

- **`PDFEditorViewModel`** — central state and business logic, observed by all views
- **`MuPDFBridge`** (Objective-C++) — thin wrapper around the MuPDF C API, bridged to Swift via a bridging header
- **`PDFTextEditEngine`** — Swift model that talks to MuPDFBridge for text block extraction and editing
- **SwiftUI views** — declarative UI layer; `PDFViewerView` hosts an `NSViewRepresentable` wrapping Apple's `PDFView`

## Distributing the App

To export a notarized app for distribution outside the Mac App Store:

1. In Xcode, set your **Team** to your **Developer ID** certificate
2. **Product → Archive**
3. In the Organizer: **Distribute App → Direct Distribution → Automatically manage signing → Export**

The hardened runtime is enabled by default (`ENABLE_HARDENED_RUNTIME = YES`). Because MuPDF is linked statically, no additional entitlements are needed.
